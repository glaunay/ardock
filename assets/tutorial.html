What is the ARbitrary DOCKing server?

ARDOCK is a web service for the study of protein surfaces. Arbitrary docking can reveal potential interaction sites on the surface of a protein using docking with a set of 25 random protein "probes". We have shown that the random probes interact in a non-random manner at protein surfaces, and that targeted regions are enriched in biological interfaces.

ARDOCK is using a PDB structure as input. The web client provides the interactive visualization of protein structures and sequences. The back-end server performs the protein surface study computations. ARDOCK uses the webGL NGL library for molecular rendering and the HeX software for fast docking computations.

Submitting and working with protein structures

Any file following the PDB standard can be submitted. In its current state, the ARDOCK server only processes natural amino-acids, any other compound will be discarded.

For multimeric proteins, each oligomeric state will give different results because the ARDOCK method is sensitive to the overall shape of the protein. If the protein you are working on contains several chains, any oligomeric state can be constructed by selectively deleting chains. If you press the tab button, a copy of the current structure will be opened in a new tab, allowing you to work on a different oligomeric state.

Each submitted protein structure file is open in a distinct tab allowing different proteins to be processed simultaneously.

For a given multimeric structure, alternative set of protein chains can be analyzed with the second tab-level (red square). Chains view can be toggled on and off using the upper right controller (yellow square). Here the chain B is hidden, and pressing the ban icon will hide the chain D.

Visualizing proteins

The upper-right part of the user's interface shows all available protein chains. Clicking on a chain identifier will display a submenu which allows to visualize the corresponding amino-acid sequence in a dedicated window. The menu also makes it possible to remove the chain from the representation, this action can be canceled until you submit the structure to the server for analysis. Residues are highlighted on the 3D structure by clicking on the structure itself or by clicking on the corresponding amino-acid letter in the sequence window.

The top-right menu (yellow rectangle) controls the molecular rendering. Three representations are currently proposed: ball-and-stick, cartoon and surface.
Another use of the sub-menu (red rectangle) is the display of each chain amino-acid sequence in a draggable window(left rectangle). Here Glutamate 57 has been selected in the sequence window, it molecular representation is colored in red and an additional textual description is displayed (green rectangle)

Submitting an Arbitrary Docking analysis

Submit a job to the server

Clicking on the GO button (red square) will send the structure to the ARDOCK server for computation. The disabled chain will be removed from the structure.

Loading time

As the task is submitted a loader is displayed to show information on its progress. Meanwhile a center console dedicated to data save and export appears.

Job restore key

For now, only the right-end button of the center console is enabled. Clicking on it will display a job identification key (yellow rectangle). The purpose of this key is to allow you to access to the job results at any time in the future. If you are forced to quit at this stage, the computations will be carried and still accessible to you.

Real time display of the computation state

All along the computation, amino acids are incrementally colored by their number of hits. The color scale is updated accordingly. The loader is also progressing.

Visualizing Results

Upon job completion, the arbitrary docking results are displayed in a table. Each row of the table stores the information relative to a single amino-acid. Clicking on an element of the table will highlight it in the molecular structure. The data table can be exported in csv format.

Arbitrary docking data table on the left end of the user's interface (yellow rectangle). The datable can be scrolled and sorted by its columns value. The table footer lays the export to csv button.

Results export and interoperabilty with the espript server

Additionally the ARDOCK server supports the export of its results in PDB format, where the b-factor column stores each amino-acid score. This annotated structure file can also be passed to the ESPRIPT server which provides many additional tools to study protein structures and sequences.

The export of the results in PDB format or their communication to the ESPRIPT server are controlled by the center console (yellow rectangle).
